# NER实体识别评估方案

## 一、评估指标体系

### 1.1 基础指标（Core Metrics）

这些指标用于评估实体识别的基本性能：

#### 1.1.1 准确率（Precision）
- **定义**：识别正确的实体数 / 识别出的实体总数
- **公式**：`Precision = TP / (TP + FP)`
- **意义**：衡量识别结果的精确程度，准确率越高，误识别越少
- **目标值**：≥ 0.90

#### 1.1.2 召回率（Recall）
- **定义**：识别正确的实体数 / 应该识别出的实体总数
- **公式**：`Recall = TP / (TP + FN)`
- **意义**：衡量识别的完整性，召回率越高，漏识别越少
- **目标值**：≥ 0.85

#### 1.1.3 F1分数（F1-Score）
- **定义**：准确率和召回率的调和平均数
- **公式**：`F1 = 2 * Precision * Recall / (Precision + Recall)`
- **意义**：综合评估识别性能，平衡准确率和召回率
- **目标值**：≥ 0.87

#### 1.1.4 准确度（Accuracy）
- **定义**：正确分类的样本数 / 总样本数
- **公式**：`Accuracy = (TP + TN) / (TP + TN + FP + FN)`
- **意义**：整体正确率
- **目标值**：≥ 0.90

### 1.2 实体级别指标（Entity-Level Metrics）

针对不同实体类型的详细分析：

#### 1.2.1 按实体类型统计
- **类型**：品牌（BRAND）、型号（MODEL）、类别（CATEGORY）、成色（CONDITION）等
- **指标**：每种类型的 Precision、Recall、F1
- **意义**：找出哪些类型的实体识别效果好，哪些需要改进

#### 1.2.2 实体边界准确率
- **精确边界匹配率**：边界完全正确的实体占比
- **部分边界匹配率**：边界有部分重叠的实体占比
- **意义**：评估分词和边界识别的准确性
- **目标值**：精确匹配率 ≥ 0.80

#### 1.2.3 标准化准确率
- **定义**：normalizedValue 正确的实体占比
- **意义**：评估实体归一化质量
- **目标值**：≥ 0.95

### 1.3 业务指标（Business Metrics）

从业务角度评估实体识别的实用性：

#### 1.3.1 覆盖率（Coverage）
- **定义**：识别出至少一个实体的查询占比
- **公式**：`Coverage = 有实体的查询数 / 总查询数`
- **意义**：衡量系统对查询的处理能力
- **目标值**：≥ 0.80

#### 1.3.2 平均识别实体数
- **定义**：每个查询平均识别出的实体数
- **公式**：`Avg = 总实体数 / 总查询数`
- **意义**：评估识别的丰富度
- **参考值**：1.5 - 2.5

#### 1.3.3 置信度分布
- **定义**：不同置信度区间的实体数量和准确率
- **区间**：[0.9-1.0], [0.8-0.9], [0.7-0.8], [0.6-0.7], [0.5-0.6], [0.0-0.5]
- **意义**：评估置信度得分的可靠性
- **要求**：高置信度区间的准确率应显著高于低置信度区间

#### 1.3.4 高置信度实体占比
- **定义**：置信度 ≥ 0.8 的实体占比
- **意义**：衡量识别结果的可信度
- **目标值**：≥ 0.75

### 1.4 性能指标（Performance Metrics）

评估系统的运行效率：

#### 1.4.1 平均处理时间
- **定义**：单个查询的平均处理耗时
- **单位**：毫秒（ms）
- **目标值**：≤ 50 ms

#### 1.4.2 P95/P99 处理时间
- **定义**：95%/99% 的请求处理时间不超过该值
- **目标值**：P95 ≤ 100 ms, P99 ≤ 200 ms

#### 1.4.3 吞吐量（Throughput）
- **定义**：每秒可处理的查询数
- **单位**：queries/second
- **目标值**：≥ 20 qps

## 二、评估方案

### 2.1 基于标注数据的评估（Ground Truth Evaluation）

**适用场景**：精确评估，用于验证系统改进效果

#### 2.1.1 数据准备
1. **抽样**：从 top10w-20251010.csv 中抽样
   - 高频查询（Top 10%）：100 条
   - 中频查询（10%-50%）：150 条
   - 低频查询（50% 以下）：150 条
   - 总计：400 条

2. **人工标注**：使用标注工具标注实体
   - 标注内容：实体类型、实体值、标准化值、边界位置
   - 标注格式：JSON 格式
   - 质量控制：双人标注，交叉验证

3. **标注数据结构**：
```json
{
  "苹果iPhone 15 Pro Max": [
    {
      "value": "苹果",
      "type": "BRAND",
      "normalizedValue": "Apple",
      "startOffset": 0,
      "endOffset": 2,
      "id": "brand_001"
    },
    {
      "value": "iPhone 15 Pro Max",
      "type": "MODEL",
      "normalizedValue": "iPhone 15 Pro Max",
      "startOffset": 2,
      "endOffset": 20,
      "id": "model_12345"
    }
  ]
}
```

#### 2.1.2 执行评估
```bash
# 运行标注数据评估
mvn test -Dtest=NerEvaluationTest#testWithGroundTruth
```

#### 2.1.3 评估输出
- 整体指标：Precision、Recall、F1、Accuracy
- 混淆矩阵：TP、FP、FN、TN
- 按实体类型统计
- 置信度分布
- 性能指标

### 2.2 基于规则的自动评估（Rule-Based Evaluation）

**适用场景**：大规模数据快速评估，无需人工标注

#### 2.2.1 评估规则
1. **词典覆盖率**：识别的实体是否在词典中
2. **置信度阈值**：高置信度实体占比
3. **实体合理性**：实体组合是否合理（如：品牌+型号）
4. **查询覆盖率**：有多少查询识别出了实体

#### 2.2.2 执行评估
```bash
# 运行自动评估
mvn test -Dtest=NerEvaluationTest#testAutoEvaluation
```

#### 2.2.3 评估输出
- 覆盖率
- 平均实体数/查询
- 高置信度实体占比
- 实体类型分布
- 平均处理时间

### 2.3 分层评估（Stratified Evaluation）

**适用场景**：分析不同查询类型的识别效果

#### 2.3.1 分层维度

**按搜索频率分层**：
- 高频查询（Top 25%）：头部热门查询
- 中高频查询（25%-50%）
- 中低频查询（50%-75%）
- 低频查询（Bottom 25%）：长尾查询

**按查询复杂度分层**：
- 简单查询：单品牌或单型号
- 中等查询：品牌+型号
- 复杂查询：品牌+型号+规格

**按实体数量分层**：
- 单实体查询
- 双实体查询
- 多实体查询

#### 2.3.2 执行评估
```bash
# 运行分层评估
mvn test -Dtest=NerEvaluationTest#testStratifiedEvaluation
```

### 2.4 典型案例分析（Case Analysis）

**适用场景**：深入分析识别效果，发现问题

#### 2.4.1 案例选择
- **成功案例**：识别效果好的查询
- **失败案例**：未识别或识别错误的查询
- **边界案例**：歧义查询、简写查询等

#### 2.4.2 执行分析
```bash
# 运行案例分析
mvn test -Dtest=NerEvaluationTest#testCaseAnalysis
```

### 2.5 实体类型覆盖率分析

**适用场景**：分析各类型实体的分布

#### 2.5.1 执行分析
```bash
# 运行覆盖率分析
mvn test -Dtest=NerEvaluationTest#testEntityTypeCoverage
```

## 三、评估流程

### 3.1 标准评估流程

```
1. 数据准备
   ├─ 加载搜索词数据（top10w-20251010.csv）
   ├─ （可选）准备标注数据
   └─ 初始化NER系统

2. 执行评估
   ├─ 运行基于标注的评估（如有标注数据）
   ├─ 运行自动评估
   ├─ 运行分层评估
   ├─ 运行典型案例分析
   └─ 运行覆盖率分析

3. 分析结果
   ├─ 查看评估报告
   ├─ 对比目标指标
   ├─ 识别问题点
   └─ 制定改进计划

4. 迭代优化
   ├─ 优化词典
   ├─ 调整规则
   ├─ 改进算法
   └─ 重新评估
```

### 3.2 快速评估（Quick Check）

```bash
# 运行典型案例分析（最快）
mvn test -Dtest=NerEvaluationTest#testCaseAnalysis

# 运行自动评估（1000条数据）
mvn test -Dtest=NerEvaluationTest#testAutoEvaluation
```

### 3.3 完整评估（Full Evaluation）

```bash
# 运行所有测试
mvn test -Dtest=NerEvaluationTest
```

## 四、标注数据准备指南

### 4.1 生成标注模板

系统会自动生成标注模板：

```bash
mvn test -Dtest=NerEvaluationTest#testWithGroundTruth
```

生成的模板文件：`src/test/resources/ner/ground_truth.json`

### 4.2 人工校验和修改

1. 打开 `ground_truth.json` 文件
2. 检查每个查询的实体标注
3. 修正错误的标注
4. 添加漏标的实体
5. 删除误标的实体
6. 保存文件

### 4.3 标注质量控制

- **双人标注**：两人独立标注，比对结果
- **抽样检查**：定期抽样检查标注质量
- **一致性检查**：确保标注标准一致
- **边界情况**：特别注意歧义和边界情况

## 六、改进建议

### 6.1 基于评估结果的改进方向

| 问题 | 改进方向 |
|------|---------|
| Precision 低 | 优化识别规则，减少误识别；提高置信度阈值 |
| Recall 低 | 扩充词典；优化分词；增加规则覆盖 |
| 某实体类型效果差 | 针对性优化该类型的词典和规则 |
| 边界匹配率低 | 优化分词算法；调整实体边界识别逻辑 |
| 覆盖率低 | 扩充词典；增加实体类型；优化识别策略 |
| 性能差 | 优化算法；使用缓存；并行处理 |

### 6.2 持续改进循环

```
评估 → 发现问题 → 制定方案 → 实施改进 → 再评估
```

定期（每周/每月）进行评估，跟踪指标变化趋势。

### 7.2 常见问题

**Q1：为什么需要分层评估？**
A：不同频率、复杂度的查询，识别难度不同。分层评估可以发现特定场景下的问题。

**Q2：没有标注数据怎么办？**
A：可以先运行自动评估和案例分析，了解基本情况。建议逐步积累标注数据。

**Q3：如何平衡 Precision 和 Recall？**
A：通过调整置信度阈值，可以在两者之间权衡。业务上更看重准确率时，可以提高阈值。

**Q4：评估频率建议？**
A：
- 开发阶段：每次重要改动后评估
- 上线前：完整评估
- 线上运行：每周快速评估，每月完整评估
