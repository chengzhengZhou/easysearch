## 评估指标详解

### 1. NDCG@10 / NDCG@5（归一化折损累积增益）

#### 作用
- 评估排序质量，位置越靠前权重越高
- 考虑相关性得分，不只看是否相关
- 归一化到 [0, 1]，便于比较

#### 计算原理
```
DCG = Σ(relevance_i / log₂(position_i + 1))
NDCG = DCG / IDCG（理想DCG）
```
- DCG：折损累积增益，位置越靠后折损越大
- IDCG：理想排序下的DCG（按相关性降序）

#### 合理值区间
- 优秀：≥ 0.80
- 良好：0.60 - 0.79
- 一般：0.40 - 0.59
- 较差：< 0.40

#### 3C商品搜索建议
- NDCG@5 ≥ 0.75（前5个结果质量）
- NDCG@10 ≥ 0.70（前10个结果质量）

#### 示例
假设查询"苹果iPhone 15 Pro Max"：
- 理想排序：相关得分 [0.95, 0.90, 0.85, 0.80, 0.75]
- 实际排序：相关得分 [0.90, 0.95, 0.75, 0.80, 0.85]
- NDCG@5 ≈ 0.92（排序略有偏差但整体相关）

---

### 2. MRR（平均倒数排名）

#### 作用
- 衡量首个相关结果的位置
- 适合“找到第一个相关结果”的场景
- 对位置敏感，第一个相关结果越靠前越好

#### 计算原理
```
MRR = (1/N) × Σ(1 / rank_i)
```
- rank_i：第i个查询中第一个相关结果的排名
- 若无相关结果，该项为0

#### 合理值区间
- 优秀：≥ 0.80（平均第1-2位出现相关结果）
- 良好：0.60 - 0.79（平均第1-2位）
- 一般：0.40 - 0.59（平均第2-3位）
- 较差：< 0.40（平均第3位之后）

#### 3C商品搜索建议
- MRR ≥ 0.70（大部分查询的第一个相关结果在前2位）

#### 示例
- 查询1：第1位相关 → 1/1 = 1.0
- 查询2：第2位相关 → 1/2 = 0.5
- 查询3：第1位相关 → 1/1 = 1.0
- MRR = (1.0 + 0.5 + 1.0) / 3 = 0.83

---

### 3. Precision@10 / Precision@5（准确率@K）

#### 作用
- 衡量前K个结果中相关结果的比例
- 关注“前K个结果有多少是相关的”
- 不考虑排序位置

#### 计算原理
```
Precision@K = (前K个结果中相关结果数) / K
```

#### 合理值区间
- 优秀：≥ 0.80（前K个中80%以上相关）
- 良好：0.60 - 0.79
- 一般：0.40 - 0.59
- 较差：< 0.40

#### 3C商品搜索建议
- Precision@5 ≥ 0.80（前5个中至少4个相关）
- Precision@10 ≥ 0.70（前10个中至少7个相关）

#### 示例
前5个结果：相关、相关、不相关、相关、相关
- Precision@5 = 4/5 = 0.80

---

### 4. Recall@10 / Recall@5（召回率@K）

#### 作用
- 衡量前K个结果覆盖了多少相关结果
- 关注“找到了多少相关结果”
- 需要知道总相关结果数

#### 计算原理
```
Recall@K = (前K个结果中相关结果数) / (总相关结果数)
```

#### 合理值区间
- 优秀：≥ 0.80（找到80%以上相关结果）
- 良好：0.60 - 0.79
- 一般：0.40 - 0.59
- 较差：< 0.40

#### 3C商品搜索建议
- Recall@5 ≥ 0.60（前5个覆盖60%以上相关结果）
- Recall@10 ≥ 0.80（前10个覆盖80%以上相关结果）

#### 示例
- 总相关结果：10个
- 前5个中相关：3个
- Recall@5 = 3/10 = 0.30（召回率较低）

---

### 5. MAP（平均精确率）

#### 作用
- 综合考虑排序和准确率
- 对每个相关结果计算其位置的精确率，再取平均
- 同时关注准确率和排序

#### 计算原理
```
AP = (1/相关结果数) × Σ(Precision@第i个相关结果的位置)
MAP = 所有查询的AP的平均值
```

#### 合理值区间
- 优秀：≥ 0.80
- 良好：0.60 - 0.79
- 一般：0.40 - 0.59
- 较差：< 0.40

#### 3C商品搜索建议
- MAP ≥ 0.75

#### 示例
排序：[相关(1), 相关(2), 不相关(3), 相关(4), 不相关(5)]
- 第1个相关：Precision@1 = 1/1 = 1.0
- 第2个相关：Precision@2 = 2/2 = 1.0
- 第3个相关：Precision@4 = 3/4 = 0.75
- AP = (1.0 + 1.0 + 0.75) / 3 = 0.92

---

### 6. 平均相关性得分（Mean Relevance Score）

#### 作用
- 反映整体相关性水平
- 使用连续得分，不只看二分类
- 用于趋势监控

#### 计算原理
```
平均相关性得分 = Σ(所有结果的相关性得分) / 结果总数
```

#### 合理值区间
- 优秀：≥ 0.75（整体相关性高）
- 良好：0.60 - 0.74
- 一般：0.45 - 0.59
- 较差：< 0.45

#### 3C商品搜索建议
- 平均相关性得分 ≥ 0.70

#### 示例
5个结果的相关性得分：[0.85, 0.80, 0.75, 0.70, 0.65]
- 平均相关性得分 = (0.85 + 0.80 + 0.75 + 0.70 + 0.65) / 5 = 0.75

## 指标选择建议

### 针对3C商品搜索的优先级

1. 核心指标（必看）
    - NDCG@5：排序质量
    - Precision@5：前5个准确率
    - MRR：首个相关结果位置

2. 辅助指标（参考）
    - MAP：综合评估
    - Recall@10：覆盖率
    - 平均相关性得分：整体水平

### 指标组合解读

| 场景 | 指标表现 | 含义 |
|------|---------|------|
| 排序好但准确率低 | NDCG高，Precision低 | 排序合理但相关结果少，需提升召回 |
| 准确率高但排序差 | Precision高，NDCG低 | 相关结果多但排序不佳，需优化排序 |
| 首个结果差 | MRR低 | 第一个结果不相关，影响用户体验 |
| 整体相关性低 | 平均相关性得分低 | 搜索结果整体质量需提升 |

### 实际应用建议

1. 日常监控：关注 NDCG@5、Precision@5、MRR
2. 算法优化：使用 MAP 和 NDCG 评估排序算法
3. 用户体验：MRR 反映“第一眼”体验
4. 业务目标：Precision 关注转化，Recall 关注覆盖率

这些指标共同评估搜索系统的相关性、排序质量和用户体验。
